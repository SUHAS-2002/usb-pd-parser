8.3.3.7.3.1
  PE_SNK_Send_Sink_Alert State
The PE_SNK_Send_Sink_Alert state Shall be entered from the PE_SNK_Ready state when the Device Policy Manager 
indicates that there is a Source alert condition to be reported.
On entry to the PE_SNK_Send_Sink_Alert state the Policy Engine Shall request the Protocol Layer to send an Alert 
Message.
The Policy Engine Shall transition to the PE_SNK_Wait_for_Get_Status State when:

The Alert Message has been successfully sent.
8.3.3.7.3.2
  PE_SNK_Wait_for_Get_Status State
On entry to the PE_SNK_Wait_for_Get_Status State the Policy Engine Shall initialize and run the 
SenderResponseTimer.
The Policy Engine Shall transition back to the PE_Give_Status State (see Figure 8.151, "Give Status State Diagram") 
when:

A Get_Status Message is received.
The Policy Engine Shall transition back to the PE_SNK_Ready (see Figure 8.133, "Sink Port State Diagram") when:

The SenderResponseTimer times out.
